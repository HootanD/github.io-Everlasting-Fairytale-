## 🚀 What Gets Deployed

### On Push to Main
```
Commit pushed
  ↓
CI/CD Pipeline runs (test, lint, build, push to registry)
  ↓ (on success)
Canary Deployment auto-triggers
  ├─ Kubernetes: 1 pod in 'canary' namespace
  └─ VPS: docker-compose with profile=canary on port 8081
  ↓
Health checks run (5 minute monitoring window)
  ├─ /health endpoint
  ├─ /ready endpoint
  └─ /metrics endpoint
  ↓
Decision: Success → Promote | Failure → Rollback
  ↓
Slack notification sent
```

### Branch Protection
```
Enforced on: main, develop
├─ Require 1 PR review
├─ Require status checks: test, build
├─ Dismiss stale reviews
├─ Require conversation resolution
├─ Prevent force pushes
└─ Prevent deletions
```

## 📋 GitHub Secrets Required

Before running workflows, set these secrets:
```
✅ DOCKER_USERNAME         (required)
✅ DOCKER_PASSWORD         (required)
✅ VPS_HOST               (required if deploying to VPS)
✅ VPS_USER               (required if deploying to VPS)
✅ VPS_SSH_KEY            (required if deploying to VPS)
✅ KUBE_CONFIG            (required if deploying to K8s)
❓ SLACK_WEBHOOK_URL      (optional, for notifications)
```

**Set via:**
```bash
# Automated
./setup-secrets.ps1 -DockerUsername ... -DockerPassword ...  # Windows
./setup-secrets.sh --docker-username ... --docker-password ...  # macOS/Linux

# Manual: GitHub UI → Settings → Secrets
```

## 📊 Deployment Architecture

```
GitHub Repository (main branch)
  ↓
CI/CD Pipeline (.github/workflows/ci-cd.yml)
  ├─ Job: test
  │  └─ npm install, npm test, linting
  ├─ Job: build
  │  ├─ Multi-platform build (amd64, arm64)
  │  ├─ Docker Scout security scan
  │  └─ Push to docker.io
  ├─ Job: deploy-kubernetes
  │  └─ kubectl set image + rollout status
  └─ Job: deploy-vps
     └─ SSH + docker-compose pull/up
        ↓
Canary Deployment (.github/workflows/canary-deployment.yml)
  ├─ Job: canary-k8s
  │  ├─ Create canary namespace
  │  ├─ Deploy 1 pod (separate from production)
  │  └─ Health checks: /health, /ready, /metrics
  ├─ Job: canary-vps
  │  ├─ Pull image, docker-compose up
  │  └─ Health checks on port 8081
  ├─ Job: canary-monitor
  │  └─ Wait 5 minutes (configurable)
  ├─ Job: canary-decision
  │  ├─ Success: Ready for promotion
  │  └─ Failure: Trigger rollback
  └─ Job: notify-canary-result
     └─ Send Slack message

Production Deployment (after canary approval)
  ├─ Kubernetes: kubectl set image on prod namespace
  └─ VPS: docker tag stable, docker-compose up app-stable
```

```
Main branch updated
  ↓

  ↓ (on success)
```

## 🎓 Learning & Customization

### Common Customizations

**Change canary monitoring duration:**
```bash
gh workflow run canary-deployment.yml -f canary_duration=15 --ref main
```

**Require 2 reviewers instead of 1:**
Edit `.github/workflows/github-branch-protection.yml`:
```yaml
required_approving_review_count: 2
```

**Add Prometheus-based auto-rollback:**
Edit `.github/workflows/canary-deployment.yml`, `canary-decision` job:
```yaml
- name: Query Prometheus
  run: |
    ERROR_RATE=$(curl 'http://prometheus/api/v1/query?query=...')
    if [[ $ERROR_RATE > 0.02 ]]; then exit 1; fi
```

**Deploy on schedule (nightly):**
```yaml
on:
  schedule:
    - cron: '0 3 * * *'  # 3am UTC daily
```
